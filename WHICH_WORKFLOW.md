# 🤔 어떤 워크플로우를 사용해야 하나요?

## 📋 두 가지 워크플로우

저장소에는 **두 가지 GitHub Actions 워크플로우**가 있습니다:

### 1️⃣ 영상만 생성 (수동 업로드)
**파일**: `.github/workflows/daily-shorts-auto.yml`  
**이름**: "🌍 Daily AI Travel Shorts - Video Generation Only"

**특징:**
- ✅ 매일 영상 자동 생성
- ✅ 이메일로 다운로드 링크 발송
- ⚠️ 유튜브 수동 업로드 필요
- 📊 자동화: **80%**

**언제 사용?**
- 유튜브 설정이 복잡해 보일 때
- 빠르게 시작하고 싶을 때
- 영상만 먼저 만들어보고 싶을 때

---

### 2️⃣ 완전 자동 (유튜브 자동 업로드) ⭐ **추천**
**파일**: `.github/workflows/youtube-auto-upload.yml`  
**이름**: "🌍 Daily AI Travel Shorts - Auto Upload to YouTube"

**특징:**
- ✅ 매일 영상 자동 생성
- ✅ 제목 자동 생성: "🌍 에펠탑 - AI 여행 쇼츠 #1216"
- ✅ 설명 자동 생성 (한글 + 해시태그)
- ✅ 태그 자동 생성
- ✅ **당신의 유튜브 채널**에 자동 업로드
- ✅ 이메일로 유튜브 링크 발송
- 📊 자동화: **100%**

**언제 사용?**
- 완전 자동화를 원할 때 (추천!)
- 30분 설정 시간을 투자할 수 있을 때
- 매일 수동 업로드가 귀찮을 때

## 🚀 워크플로우 전환 방법

### 현재 상태 확인

```bash
# GitHub 저장소 → Actions 탭 확인
# 어떤 워크플로우가 활성화되어 있는지 확인
```

### 워크플로우 1 → 워크플로우 2로 전환

**Step 1: 워크플로우 1 비활성화**

1. `.github/workflows/daily-shorts-auto.yml` 파일 열기
2. 10-11번 줄 확인:
```yaml
  # schedule:
  #   - cron: '0 0 * * *'
```
3. 이미 주석 처리되어 있으면 OK! (비활성화 상태)

**Step 2: 워크플로우 2 활성화**

1. `.github/workflows/youtube-auto-upload.yml` 파일 열기
2. 5-6번 줄 확인:
```yaml
  schedule:
    - cron: '0 0 * * *'
```
3. 주석이 없으면 OK! (활성화 상태)

**Step 3: YouTube 설정**

📘 **상세 가이드**: `100_PERCENT_AUTOMATED.md` 파일 참조

**간단 요약:**
1. Google Cloud 프로젝트 생성 (10분)
2. YouTube Data API 활성화 (3분)
3. OAuth 인증 (10분)
4. `python3 scripts/youtube_auth.py` 실행 (5분)
5. GitHub Secrets에 `YOUTUBE_TOKEN_BASE64` 추가 (2분)

**총 30분 (1회만)**
